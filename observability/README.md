# Observability Directory (`observability/`)

## What is the use of this folder?
This folder contains the infrastructure-as-code and configuration files for monitoring, logging, and metrics aggregation.

## What it will do
It houses the configuration for the ELK stack (Elasticsearch, Logstash, Kibana) and Allure dashboards. It takes the raw logs generated by the test suites and formats them so they can be ingested and visualized on real-time dashboards.

## How to use it
- Update `logstash/pipeline/logstash.conf` to add new grok patterns or adjust strict schema validation.
- Run `docker-compose up -d` from the root project to spin up the observability stack defined by these configs.
- Kibana dashboards and visualization templates should be exported and stored here for backup.
