# Documentation Directory (`docs/`)

## What is the use of this folder?
This folder stores all architectural diagrams, governance rules, API contracts, and high-level platform documentation.

## What it will do
It acts as the single source of truth for the QA platform's design decisions, acting as an onboarding guide for new engineers.

## How to use it
- Write markdown files, PlantUML diagrams, or architecture decision records (ADRs) and store them here.
- Share this folder with new team members so they can understand the separation between `qe-core`, `qe-ui`, etc.
